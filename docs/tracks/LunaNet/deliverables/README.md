# LunaNet LSIS-AFS Deliverables Planning Pack

This folder contains execution-ready LunaNet plans organized for a 6-person team across Gateways 0-8.

## Documents

- Program plan: [LunaNet-Deliverables-Program-Plan.md](./LunaNet-Deliverables-Program-Plan.md)
- Phase 1 plan (Gateways 0-3): [Phase-1-Foundation-and-Encoding.md](./Phase-1-Foundation-and-Encoding.md)
- Phase 2 plan (Gateways 4-6): [Phase-2-Signal-and-Decoding.md](./Phase-2-Signal-and-Decoding.md)
- Phase 3 plan (Gateways 7-8): [Phase-3-Integration-Submission.md](./Phase-3-Integration-Submission.md)
- Gateway RACI matrix: [Gateway-RACI-Matrix.md](./Gateway-RACI-Matrix.md)

## Team Assumptions

- Team size: 6
- Roles: 3 engineers + 3 developers
- Objective: pass minimum viability gate and maximize score through evidence-driven execution.

## Why This Structure

- The phase split follows dependency order: code families and encoding first, then signal and decoding, then integration and package hardening.
- RACI is separated into its own file to keep ownership stable even when phase tasks evolve.
- Program-level and phase-level plans are separated so schedule governance and implementation detail can evolve independently.

## Usage

1. Assign names to role placeholders and confirm role backups.
2. Run phase plans as gateway checklists with weekly review against acceptance criteria.
3. Keep RACI and evidence links current at each gateway close.
